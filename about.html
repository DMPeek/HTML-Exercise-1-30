Definitions for learned terminology

Disclaimer: will be using my own definitions so this will be fairly informal to practice my own understandings.

Github

  • This is the site used to host repositories that coders use for projects.
  • This is also a good site to help build a portfolio from since you can track projects.

Repo

  • These are the storage spaces for the code made for projects.
  • These can be posted and viewed publicly or can be made private.

VSCode

  • This is a code editor that is used to edit a variety of file types.
  • This can be used in tandem with github desktop to make changes to repositories.

Control Panel

  • This is a terminal that can be used to enter Git Commands.
  • This will be how you commit and finalize changes to your projects. Github Desktop can streamline some of the process.

Git Commands

  • Commands entered into terminals to execute a variety of actions for a repository.
  • Example actions are as follows: Git Add, Git Commit, Git Push, Git Checkout, Git Init, Git Clone, and Git Pull.

Markdown

  • A Type of file used to help display and read text on an output device.
  • Code used here will be in a different format than things like html, css, or js files

Servers

  • These are computers connected to the internet that are intended to serve a purpose once connected to.
  • The types of servers are as follows: Fileserver to Serve/Offer files, Webserver to offer websites to visit, and Database Server to host a database to access.
